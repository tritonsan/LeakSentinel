# Budget Plan ($75 Existing AWS Credits)

Goal: Prove every claimed AWS integration with real APIs (not mocks), while keeping total spend safely under $75 until we get additional credits.

## Budget Envelope
- Total available now: $75
- Hard stop (do not exceed): $60 (leave buffer for surprises)
- Buffer: $15

## Cost Drivers (What Will Burn Credits Fast)
- Always-on hosting (ECS/Fargate running 24/7).
- Load balancer running 24/7 (ALB has hourly cost even at zero traffic).
- NAT Gateway (hourly + data processing). This can destroy a small credit budget quickly.
- Excessive Bedrock calls (large prompts/responses, long audio streaming sessions).
- Chatty logging (CloudWatch ingestion + retention).

## Strategy (How We Stay Under Budget)
1. Local-first development (default).
   - Build and debug the workflow locally using deterministic scenarios.
   - Use AWS only for short, intentional integration tests.
2. Bedrock calls are time-boxed and measured.
   - Run a small set of smoke tests to prove Nova integration.
   - Cache results where possible; avoid repeated "same input" runs.
3. Hosted stack is "on only when needed".
   - Deploy only for demo rehearsal / video capture windows.
   - Tear down the stack immediately after.
4. Avoid NAT Gateway costs during hackathon.
   - If we need outbound AWS service access from ECS tasks, prefer public subnets + public IP for tasks (hackathon acceptable) OR use proper VPC endpoints later if we must stay private.

## Allocation (Practical Split)
- $40: Amazon Bedrock (Nova) integration tests
  - Nova 2 Lite (reasoning/decision synthesis)
  - Nova Multimodal Embeddings (similar-incident retrieval)
  - Nova 2 Sonic (voice streaming) - short sessions only
- $15: Hosted demo infra tests (ECS Fargate + ALB + CloudWatch logs)
  - Only for limited windows; no 24/7
- $5: ECR pushes / misc AWS overhead
- $15: Buffer (do not plan to spend)

## “Proof We Use Real APIs” Test Matrix (Minimum)
These are the minimum non-mock checks we must be able to run and screenshot/log for submission credibility.

- Bedrock / Nova 2 Lite
  - One end-to-end run where the final decision text + rationale is generated by Nova (not local fallback).
- Bedrock / Nova Multimodal Embeddings
  - One retrieval query against our incident memory using Nova embeddings.
- Bedrock / Nova 2 Sonic (optional for early submit, required if we claim voice)
  - One short push-to-talk interaction or a short audio-file upload round-trip.
- Nova Act (optional; required only if we claim it in the demo)
  - One scripted interaction (e.g., open Ops Portal, filter planned ops, return result).

## Guardrails (Non-Negotiable Rules)
- No always-on ALB/Fargate for days. If not actively demoing, the stack must be deleted.
- If spend hits $50, pause all hosting and only run Bedrock smoke tests that are strictly necessary.
- If spend hits $60, stop all AWS usage until we have more credits.

## Operational Controls (Do This Immediately)
- Create an AWS Budget alert at $50 and $60.
- Tag all hackathon resources with `Project=LeakSentinel` and `Env=Hackathon`.
- Set CloudWatch Logs retention low (e.g., 3-7 days) for hackathon stacks.

## Deployment Note (Avoid NAT)
Our ECS template supports assigning a public IP to tasks (see `infra/cfn/ecs-fargate-leaksentinel.yaml`).
For a low-cost hackathon deployment:
- Run ALB in public subnets.
- Run ECS tasks in public subnets with `AssignPublicIp=ENABLED`.
- Security group still restricts inbound to ALB -> task only.

This avoids NAT Gateway costs while still proving real AWS integration.
